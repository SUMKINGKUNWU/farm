- 交易密码设置与校验。

## 本次完成

本次实现交易站 MVP 即时买卖：

- 交易站买入收获物。
- 交易站卖出收获物。
- 交易密码校验。
- 交易站 3% 税费。
- 写入交易记录。
- 写入税收记录。
- 写入金币资产流水。
- 写入商品资产流水。

本次暂不实现订单簿撮合，先用系统即时成交跑通市场闭环。

## 新增接口

### 买入

```text
POST /api/users/{userId}/market/buy
```

请求：

```json
{
  "itemCode": "WHEAT",
  "quantity": 100,
  "tradePassword": "654321"
}
```

规则：

- 只能买入 `HARVEST` 收获物。
- 单价来自 `items.current_price`。
- 交易额由服务端计算。
- 买入总成本 = 成交额 + 税费。
- 税费按 `tax_config.MARKET` 计算，当前 3%。
- 买入成功后扣金币、加库存。

示例：

```text
小麦单价 25
买入数量 100
成交额 2,500
税费 75
总成本 2,575
```

### 卖出

```text
POST /api/users/{userId}/market/sell
```

请求：

```json
{
  "itemCode": "WHEAT",
  "quantity": 40,
  "tradePassword": "654321"
}
```

规则：

- 只能卖出 `HARVEST` 收获物。
- 玩家库存必须足够。
- 卖出到账 = 成交额 - 税费。
- 卖出成功后扣库存、加金币。

示例：

```text
小麦单价 25
卖出数量 40
成交额 1,000
税费 30
到账 970
```

## 数据落点

交易成功后写入：

- `market_trades`：成交记录。
- `tax_records`：税收记录。
- `asset_ledger`：金币和商品流水。
- `wallets`：钱包余额。
- `player_inventory`：商品库存。

## 事务一致性

本次实现保护：

- 交易密码校验通过后才执行交易。
- 商品价格只从服务端读取。
- 税率只从服务端读取。
- 钱包使用 `for update` 锁定。
- 钱包更新带 `version` 条件。
- 买入库存使用 `on conflict` 原子累加。
- 卖出库存使用 `available_quantity >= quantity` 条件扣减。
- 交易记录、税收记录、资产流水与钱包库存变更在同一事务内提交。
