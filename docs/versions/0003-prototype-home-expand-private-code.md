# 0003 - 首页简化、扩建地块与私下交易验证码

## 当前已有

上一阶段已经完成静态页面原型：

- 总览大厅。
- 农场牧场。
- 交易站。
- 私下交易。
- 管理后台。

原型文件位于：

- `prototypes/farm-exchange/index.html`

## 本次完成

本次根据页面确认反馈调整原型：

- 首页主标题由偏海报化的大字改为更克制的经营摘要。
- 首页增加可收获、待成交、风控状态三个摘要指标。
- 农场牧场页明确土地和牧栏都可以扩建。
- 地块中增加“扩建土地”和“扩建牧场”状态。
- 投产面板补充扩建费用递增和产能通胀提示。
- 私下交易页增加双方验证码输入。
- 私下交易成交条件调整为验证码、双方确认、库存锁定、金币锁定和税费预扣共同成立。

## 产品范围

本次重点确认三个产品方向：

- 首页应该更像经营看板，而不是宣传页。
- 土地和牧场栏位需要具备持续扩建能力，作为长期经营目标和金币回收点。
- 私下交易不能只靠双方点击确认，需要增加双方约定验证码，降低误操作、骗单和冒名确认风险。

## 技术范围

本次仍为静态 HTML 原型调整。

后续开发时需要注意：

- 扩建土地和扩建牧场需要独立配置扩建成本曲线。
- 扩建成本建议由后台配置，不写死在前端。
- 私下交易验证码不能明文长期保存。
- 验证码应设置有效期和最大尝试次数。
- 验证码匹配不能绕过库存、金币、税收和大宗令牌校验。

## 风险与漏洞

私下交易验证码可以降低误确认风险，但不能单独视为安全机制。

仍需防护：

- 验证码暴力尝试。
- 验证码泄露。
- 创建报价后价格剧烈变化。
- 对方确认时库存或金币不足。
- 大宗令牌在确认前被消耗。
- 双方同时修改报价导致状态不一致。

建议私下交易状态机：

```text
DRAFT -> WAIT_COUNTERPARTY -> CODE_PENDING -> LOCKING_ASSETS -> CONFIRMED -> SETTLED
                                   |                 |
                                   v                 v
                                CANCELLED          FAILED
```
