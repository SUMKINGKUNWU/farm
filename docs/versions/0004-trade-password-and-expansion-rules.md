# 0004 - 交易密码与栏位扩建规则

## 当前已有

上一阶段原型已经包含：

- 简化后的首页经营摘要。
- 农场牧场扩建地块。
- 私下交易确认机制。
- 静态页面截图输出。

## 本次完成

本次根据反馈继续调整原型和规则：

- 私下交易的双方验证码改为交易密码。
- 玩家只要开始使用交易功能，就必须先设置交易密码。
- 私下交易由双方各自输入自己的交易密码确认。
- 初始用户资金设为 10,000。
- 土地和牧场栏位最多都可以扩建到 16 个。
- 每扩建下一个栏位，需要的资金按翻倍规则增长。
- 农场牧场页展示当前栏位数量、最大栏位数量和下一次扩建费用。

## 产品范围

交易密码属于账户级安全能力，不只服务私下交易。

建议适用范围：

- 交易站挂单。
- 交易站买入或卖出。
- 私下交易创建。
- 私下交易确认。
- 大宗交易令牌使用。
- 高风险资产操作。

首次使用交易能力时，如果玩家未设置交易密码，应跳转或弹出交易密码设置流程。

## 扩建规则

初始规则：

```text
新用户初始资金 = 10,000
土地最大数量 = 16
牧栏最大数量 = 16
下一栏位费用 = 当前栏位扩建费用 * 2
```

示例：

```text
第 1 次扩建 = 1,000
第 2 次扩建 = 2,000
第 3 次扩建 = 4,000
第 4 次扩建 = 8,000
第 5 次扩建 = 16,000
```

实际起始扩建费用建议后台可配置，土地和牧栏也可以分别配置不同基础费用。

## 技术范围

后续后端需要增加：

- 用户交易密码状态字段。
- 交易密码设置接口。
- 交易密码校验接口或交易确认时内联校验。
- 交易密码错误次数限制。
- 交易密码锁定时间。
- 扩建配置表。
- 玩家土地数量和牧栏数量字段。
- 扩建交易流水和资产流水。

交易密码不能明文保存，必须使用安全哈希。

## 风险与漏洞

需要重点防护：

- 交易密码暴力破解。
- 密码错误次数无限尝试。
- 前端绕过未设置交易密码状态。
- 同时多次点击扩建导致重复扩建。
- 余额扣除成功但栏位未增加。
- 栏位数量超过 16。
- 修改前端价格绕过扩建费用。

扩建必须以后端事务为准：

```text
校验当前栏位 < 16
计算服务端扩建费用
校验余额足够
扣减金币
增加栏位
写入资产流水
提交事务
```

## 下次待做

建议下一步确认：

- 土地和牧栏是否分别从几个初始栏位开始。
- 第一次扩建费用是多少。
- 交易密码是否用于所有交易，还是只用于私下交易和大宗交易。
- 是否需要增加“设置交易密码”弹窗原型。
- 是否需要新增“仓库页”和“订单簿页”。

确认后可进入下一阶段：

- 继续调整原型；或
- 初始化 Vue 3 + Vite 工程并把当前原型迁移为组件。
