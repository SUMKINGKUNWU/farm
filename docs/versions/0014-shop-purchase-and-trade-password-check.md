新增能力：

- 交易密码校验服务。
- 商店购买接口。
- 购买种子和动物。

- 购买时增加库存。

- 交易密码错误返回 403。
- 收获物禁止从商店直接购买。

## 新增接口

### 商店购买

```text
POST /api/users/{userId}/shop/purchase
```

请求：

```json
{
  "itemCode": "WHEAT_SEED",
  "quantity": 3,
  "tradePassword": "654321"
}
```

响应：

```json
{
  "itemCode": "WHEAT_SEED",
  "quantity": 3,
  "unitPrice": 80,
  "totalAmount": 240,
  "balanceAfter": 9760,
  "availableQuantityAfter": 3
}
```

## 业务规则

商店当前只允许购买：

- `SEED`：种子。
- `ANIMAL`：动物或幼崽。

商店当前不允许购买：

- `HARVEST`：收获物。
- `TOKEN`：令牌。
- `FEED`：饲料，后续可开放。
- `CONSUMABLE`：消耗品，后续可开放。

原因：

- 收获物应主要来自种植、养殖、交易站或玩家私下交易。
- 避免商店直买收获物破坏经营和市场价值。

## 交易密码规则

本次新增：

- `TradePasswordService`

当前规则：

- 未设置交易密码时，禁止执行购买。
- 交易密码错误时，失败次数加 1，并返回 403。
- 交易密码正确时，清空失败次数和锁定时间。
- 交易密码使用 BCrypt 校验，不明文比较。

后续仍需增强：

- 错误次数达到阈值后锁定。
- 锁定时间配置化。
- 高风险交易二次确认。

## 事务一致性

购买接口在同一事务中完成：

```text
校验交易密码
查询商品
校验商品可购买
锁定钱包
校验余额
扣除金币
增加库存
写金币流水
写商品流水
提交事务
```

关键保护：

- 价格由服务端读取 `items.current_price`。
- 总金额由服务端计算，不接收前端金额。
- 钱包使用 `for update` 锁定。
- 钱包更新带 `version` 条件。
- 库存使用 `on conflict` 原子累加。
