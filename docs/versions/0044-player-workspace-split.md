# 0044 玩家端工作区拆分

## 当前已有

- 玩家端页面已具备农田与牧场、农场商店、交易站、私下交易、底部页签和玻璃拟态悬浮信息层。
- 玩家端支持交易密码前置提示、栏位扩建、库存查看、行情刷新、交易税费预估和大宗令牌输入。
- 管理台已在上一版拆为税率、令牌、资产、交易记录 4 个子面板。

## 本次完成

- 将 `GamePlayerWorkspace.vue` 从单个大组件拆分为组合容器。
- 新增玩家端子组件：
  - `GameTopbar.vue`：顶部用户、金币、可收获数量和刷新入口。
  - `FarmRanchPanel.vue`：农田、牧场、扩建、播种、养殖和收获。
  - `ShopPanel.vue`：种子与动物购买、库存展示。
  - `MarketPanel.vue`：交易站买入、卖出、税费预估和价格规则说明。
  - `MarketSidebar.vue`：行情侧栏、税费规则和令牌数量。
  - `PrivateTradePanel.vue`：私下交易报价、接受和取消入口。
  - `GameBottomTabs.vue`：底部游戏页签。
  - `PlayerFloatLayer.vue`：个人信息、物品信息和交易确认悬浮层。
- 父组件继续保留表单状态、计算属性和后端动作调用，子组件只负责展示和触发父级动作。
- 保留 Esc 关闭悬浮层和打开后聚焦关闭按钮的可访问性行为。

## 验证结果

- 已执行 `npm.cmd run build`，前端生产构建通过。
- 当前会话未暴露可用的 in-app browser 操作工具，未执行浏览器页面冒烟截图。
- 构建仍提示 ECharts 独立 chunk 超过 500 kB；该 chunk 由管理台交易记录图表动态加载。

## 下次建议

- 对拆分后的玩家端做浏览器交互回归，重点检查底部页签切换、悬浮层打开关闭、交易密码输入和表单提交。
- 继续优化组件通信方式，可逐步把函数型 props 调整为 `emit` 事件，降低子组件对父组件实现细节的依赖。
