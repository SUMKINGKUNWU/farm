# 0026 玩家端网页 MVP

## 当前已有能力

- 后端已提供玩家登录态接口 `/api/me/*`，玩家操作不再需要前端手填用户 UUID。
- 前端已有管理台、统一错误处理、错误码映射和结构化提示。
- 管理台仍支持税率、令牌、玩家资产、交易记录查询。

## 本次完成内容

- 前端新增玩家端/管理台模式切换，保留同一套农场经营视觉语言。
- 新增 `playerStore`，玩家端接口统一走 Pinia 管理，并复用统一错误处理。
- 玩家端支持注册、登录、自动恢复 token、退出登录。
- 玩家端支持读取概览数据：
  - 可用金币
  - 锁定金币
  - 农田栏位
  - 牧场栏位
  - 下次扩建费用
  - 交易密码是否已设置
- 玩家端支持设置 6 位交易密码。
- 玩家端支持商店购买种子和动物，并刷新库存。
- 玩家端支持农田/牧场扩建。
- 玩家端支持在第一个空闲农田或牧场栏位开始生产。
- 玩家端支持记录最近生产批次，并用最近批次或手动输入 `growthId` 发起收获。
- 玩家端支持交易站行情查询、买入、卖出和大宗令牌字段输入。
- 玩家端支持查看库存和当前可用大宗令牌数量。

## 风险与说明

- 本轮玩家端是 MVP 操作台，不是最终游戏 UI。
- 私下交易需要买卖双方协作和报价单管理，本轮暂未做完整 UI。
- 当前生产收获主要依赖最近生产批次或手动输入 `growthId`，后续应增加后端“进行中生产列表”接口，让页面自动展示可收获批次。
- 农田/牧场栏位目前只展示第一个空闲栏位，后续应做成 16 格可视化地块。

## 下次建议目标

- 补充后端“我的生产批次列表”接口，并在玩家端展示所有进行中/可收获批次。
- 将农田和牧场做成最多 16 格地块视图，支持按格子种植、养殖、收获。
- 增加私下交易 UI：创建报价、输入买方用户 ID、接受报价、取消报价、大宗令牌校验。
