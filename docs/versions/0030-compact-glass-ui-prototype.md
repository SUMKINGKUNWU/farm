# 0030 紧凑玻璃拟态界面原型

## 当前已有内容

- 已有 5 套前端视觉 SVG 原型，覆盖木牌农场、经营桌面、集市交易镇、柔和移动端农场、柔和交易站。
- 用户确认整体方向可以继续走柔和游戏化界面，但指出上一版整体比例偏大。
- 用户明确希望玻璃拟态用于个人信息、用户信息、物品信息等悬浮信息展示，更适合小弹窗。

## 本次完成内容

- 新增紧凑版玻璃拟态原型：`prototypes/farm-exchange/ui-style-f-compact-glass-farm.svg`。
- 将主界面比例调整为 3 个手机游戏画布并排展示，避免大屏后台式面板感。
- 农田和牧场地块缩小为紧凑栏位，保留最多扩建 16 个栏位的表达。
- 将玻璃拟态限定为悬浮层使用：
  - 个人信息与资产信息悬浮卡。
  - 物品详情悬浮卡。
  - 私下交易确认悬浮卡。
- 主界面使用更稳定的纸感、草地、土地、金币色块承载玩法信息，降低透明层对可读性的影响。
- 私下交易确认继续保留交易密码概念，表达双方确认、库存锁定、资金锁定、税费确认的流程。

## 设计判断

- 主界面不建议大量使用玻璃拟态，因为农场游戏需要长期查看地块状态、库存和数字，透明背景过多会削弱识别效率。
- 玻璃拟态适合承载“临时出现、需要聚焦”的信息，例如点击头像、点击商品、点击交易确认时弹出的浮层。
- 页面比例应优先面向网页端中的移动游戏式布局，后续可以在桌面端外层增加背景、侧栏或多窗口，但核心操作区仍保持紧凑。

## 下次建议

- 如果用户确认该方向，下一步把 Vue 玩家端首页改成紧凑 HUD 结构。
- 优先落地农场、商店、交易站、私下交易 4 个底部页签。
- 先做静态视觉改造，再接入现有 Pinia 数据和接口，避免一次性改动过大。
