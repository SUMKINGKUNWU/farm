# 0002 - 静态页面原型

## 当前已有

上一阶段已经建立本地文档体系，并完成第一版产品与技术落地方案。

当前已明确：

- 首发形态是网页版本。
- 技术栈选用 Vue 3 + Vite + Pinia + ECharts、Spring Boot、PostgreSQL、Redis。
- 文档采用只增不改策略，每次完成阶段后新增一份版本文档。

## 本次完成

本次新增静态页面原型，用于先确认页面效果和功能信息架构。

新增文件：

- `prototypes/farm-exchange/index.html`

原型特点：

- 单文件静态 HTML，不依赖外部 CDN。
- 包含总览大厅、农场牧场、交易站、私下交易、管理后台五个页面状态。
- 通过左侧导航切换页面。
- 使用“农场账本 + 商品交易所”的视觉方向。
- 包含交易税率、私下交易税率、大宗令牌、风控事件、管理员审计等核心概念。
- 使用内置 SVG 模拟行情图，后续 Vue 版本再替换为 ECharts。

## 产品范围

本次原型覆盖以下页面：

- 总览大厅：展示金币、仓库、大宗令牌、市场涨跌、成长资产、今日任务和风控提示。
- 农场牧场：展示土地网格、牧栏、空地播种、成熟收获、投产成本和预计收益。
- 交易站：展示商品列表、价格趋势、成交量、买卖面板、税费计算和订单入口。
- 私下交易：展示玩家报价单、私下交易税、市场参考价、大宗令牌和风险提示。
- 管理后台：展示税率、大宗阈值、市场熔断、风控事件和审计日志。

## 技术范围

本次仅做静态原型，不引入 Vue 工程。

原因：

- 先确认页面和功能，不急于进入工程化。
- 单文件更方便迁移和截图。
- 视觉方向确定后再拆成 Vue 组件，避免过早工程化导致返工。

后续迁移到 Vue 时，建议拆分：

- `AppLayout.vue`
- `SidebarNav.vue`
- `DashboardPage.vue`
- `FarmPage.vue`
- `MarketPage.vue`
- `PrivateTradePage.vue`
- `AdminPage.vue`
- `MarketChart.vue`
- `StatCard.vue`
- `TradeForm.vue`

## 风险与漏洞

本次原型不包含真实业务逻辑，因此以下能力仍未实现：

- 后端事务。
- 数据库锁。
- 库存一致性。
- 税收入账。
- 大宗令牌真实校验。
- 价格动态计算。
- 交易并发处理。
- 管理员真实权限控制。

这些风险已在页面中通过提示信息体现，后续开发时必须落到后端。

## 下次待做

建议下一步先基于该原型做确认：

- 页面风格是否符合预期。
- 五个主页面是否足够。
- 是否需要更偏游戏化，还是更偏交易所。
- 农场牧场是否要分成两个独立页面。
- 交易站是否在 MVP 就做订单簿，还是先做即时买卖。
- 后台管理是否需要在 MVP 阶段就完整做出。

确认后建议新增 `0003` 版本文档，并推进以下方向之一：

- 方向 A：按当前原型初始化 Vue 3 + Vite 项目并迁移页面。
- 方向 B：继续打磨高保真原型，增加登录页、商品详情页、订单簿和仓库页。
- 方向 C：先做数据库 ER 图和 Spring Boot 后端骨架。
