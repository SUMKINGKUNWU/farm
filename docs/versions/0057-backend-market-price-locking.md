# 0057 后端交易站行情价格行级锁

## 当前已有

- 交易站行情并发快照测试已覆盖并发成交时成交记录、成交量和快照不会丢失。
- 上一版测试发现当前价在并发买入后可能为 26 或 27，说明价格更新还没有严格累积每笔成交影响。

## 本次完成

- 修改 `MarketService.marketItem`，交易入口读取商品时使用 `for update` 锁定当前商品行。
- 成交价、税费和行情新价格都基于锁内最新 `current_price` 计算，避免两笔并发成交都基于同一个旧价格写回。
- 统一锁顺序为先锁商品行、再处理用户钱包和库存，避免不同用户并发交易同一商品时出现商品锁和钱包锁交叉死锁。
- 收紧并发行情测试，将两笔并发买入后的最终价从 `26 到 27` 的边界断言改为稳定等于 `27`。

## 影响范围

- 同一商品的并发成交会按商品行串行处理，第二笔成交会读取第一笔更新后的价格。
- 钱包扣款、税费和库存入账继续沿用原有事务内落库逻辑。
- 高并发下同一商品价格更新会更稳定，但同一商品成交会在价格更新阶段发生短暂排队。

- 后续可评估是否把成交价也改为锁内最新价格，以实现更接近实时撮合的行情体验。
