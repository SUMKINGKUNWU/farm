# 0011 - Git 初始化与远程仓库接管

## 当前已有

当前项目已经具备：

- 产品与版本文档体系。
- 静态页面原型。
- Spring Boot 后端骨架。
- PostgreSQL/Flyway 数据库迁移。
- 用户注册、初始化资产、交易密码接口。

## 本次目标

本次将项目交给 Git 管理，并推送到远程仓库：

```text
git@github.com:SUMKINGKUNWU/farm.git
```

## 本次新增

新增 `.gitignore`，避免提交：

- Maven 依赖缓存。
- Java 编译产物。
- 本地日志。
- 编辑器配置。
- 本地密钥和环境覆盖文件。

## 后续 Git 管理规则

后续每完成一个阶段：

- 新增一份版本文档。
- 提交代码和文档。
- 提交信息说明本阶段完成内容。
- 推送到远程仓库。

## 注意事项

当前 `application.yml` 仍包含本地开发默认数据库密码。后续建议改为仅使用环境变量或提供不含真实密码的示例配置。
