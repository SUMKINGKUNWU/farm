# 0020 Vue 管理端联调页面

## 当前已有内容

- 后端已支持玩家经营主线、交易站、私下交易、大宗交易令牌和管理端基础接口。
- 管理端后端接口包含税率配置、管理员发放大宗令牌、玩家资产查询和玩家交易记录查询。
- 静态原型已确定整体视觉方向：深绿侧栏、暖色纸张质感、农场经营台风格。

## 本次完成内容

- 新增正式前端工程 `frontend/`：
  - Vue 3
  - Vite
  - Pinia
  - ECharts
- 新增 Vite 代理：
  - 前端请求 `/api` 自动代理到 `http://127.0.0.1:3000`。
- 新增管理端单页控制台：
  - 管理员用户 ID 输入。
  - 目标玩家用户 ID 输入。
  - 税率配置读取与修改。
  - 大宗交易令牌发放。
  - 玩家资产查看。
  - 玩家交易记录查看。
  - 交易金额与税费 ECharts 图表。
- 新增 Pinia 管理端 store：
  - 统一管理管理端接口请求。
  - 统一处理 loading、成功提示和错误提示。
- 更新 `.gitignore`：
  - 忽略 `frontend/node_modules/`。
  - 忽略 `frontend/dist/`。
- 新增 `package-lock.json`，固定前端依赖版本。

## 验证结果

- 已执行前端生产构建：
  - `npm run build`
  - 构建通过。
- 构建时出现 ECharts 打包体积提示：
  - 当前管理台直接引入 ECharts，构建 chunk 超过 Vite 默认提示阈值。
  - 这不影响运行，后续可通过懒加载或 manualChunks 拆包优化。
- `npm install` 输出显示依赖已安装完成，但命令在收尾阶段超时退出；随后 `npm run build` 成功，说明依赖可用。

## 风险与后续注意

- 当前管理端页面仍使用 `adminUserId` 查询参数作为临时管理员校验，正式上线前必须替换为登录态/token。
- 当前页面是管理端 MVP 单页，还没有路由、登录页、权限菜单和分页筛选。
- 交易记录图表暂时基于最近返回数据渲染，没有做时间范围筛选。
- 前端尚未与真实管理员账号创建流程联动，需要后续提供管理员初始化或后台创建入口。

## 下次建议目标

- 优先实现正式登录态/token 鉴权，让玩家端和管理端都不再依赖 URL 中传入用户 ID。
- 或继续扩展前端玩家侧页面，把注册、交易密码、农场、商店、交易站和私下交易联调成完整 MVP 操作流。
