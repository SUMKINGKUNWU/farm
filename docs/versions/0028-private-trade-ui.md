# 0028 私下交易 UI 与我的报价列表

## 当前已有能力

- 玩家端已具备登录、概览、交易密码、商店、生产、收获、交易站和 16 格地块视图。
- 后端已支持私下交易创建、接受、取消和 5% 税收。
- 大宗交易令牌已可在交易站和私下交易接受时校验。

## 本次完成内容

- 后端新增 `PrivateTradeOfferResponse`，用于展示报价列表。
- 后端新增我的私下交易报价列表接口：
  - `GET /api/users/{userId}/private-trades`
  - `GET /api/me/private-trades`
- 报价列表返回买卖双方用户、商品、数量、总价、税费、状态、过期时间、成交/取消时间、创建时间。
- 测试覆盖卖方创建报价后双方都能查询报价列表。
- 测试覆盖取消后列表状态变为 `CANCELLED`。
- 测试覆盖接受后列表状态变为 `COMPLETED`。
- 玩家端 dashboard 会同步加载私下交易报价。
- 玩家端新增私下交易区域：
  - 创建卖出报价
  - 输入买方用户 ID
  - 选择收获物商品
  - 输入数量、总价、卖方交易密码
  - 买方配置接受报价所需交易密码和可选大宗令牌
  - 报价列表展示我作为卖方或买方的相关报价
  - 买方可接受 `WAIT_ACCEPT` 报价
  - 卖方可取消 `WAIT_ACCEPT` 报价

## 风险与说明

- 当前私下交易仍需要卖方输入买方用户 ID，后续应增加玩家搜索或好友/联系人能力。
- 接受报价使用统一的接受配置区，后续可改为每条报价弹窗确认，降低误操作风险。
- 报价列表当前限制最近 100 条，后续应增加分页、状态筛选和“只看待处理”。

## 下次建议目标

- 增加玩家搜索/用户查找接口，支持通过用户名找到买方用户 ID。
- 私下交易增加确认弹窗，显示买方总成本、卖方到账、税费和令牌要求。
- 优化前端 ECharts 打包体积，避免玩家端也加载管理台图表依赖。
