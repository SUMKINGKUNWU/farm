# 0018 大宗交易令牌校验

## 当前已有内容

- 后端已支持用户注册、交易密码、经营摘要、田地/牧栏扩建、种植养殖收获、商店购买、交易站买卖、行情快照、私下交易报价单。
- 数据库已预留 `bulk_trade_tokens` 表，并在 `items` 中配置了 `bulk_quantity_threshold` 与 `bulk_amount_threshold`。
- 交易站税率为 3%，私下交易税率为 5%，均通过 `tax_config` 配置。

## 本次完成内容

- 新增大宗交易令牌模块：
  - `GET /api/users/{userId}/bulk-tokens` 查询玩家令牌。
  - `POST /api/users/{userId}/bulk-tokens` MVP 发放令牌。
  - 支持令牌商品类型限制、单笔限额、总限额、剩余次数、过期时间和状态。
- 交易站买卖接入大宗交易校验：
  - 当交易数量达到商品大宗数量阈值，或交易金额达到商品大宗金额阈值时，必须提交 `bulkTokenCode`。
  - 成交前校验令牌归属、状态、过期时间、商品类型、单笔限额、总限额。
  - 成交时扣减令牌次数、累计使用金额，用尽后置为 `USED`。
- 私下交易接入大宗交易校验：
  - 报价创建阶段仍冻结卖方库存。
  - 买方接受报价时按报价数量和金额判断是否大宗。
  - 大宗报价由买方提交 `bulkTokenCode` 并在结算成功时扣减令牌。
  - 成交报价单记录使用过的 `bulk_token_id`。
- 补充后端测试：
  - 交易站大宗买入无令牌被拒绝。
  - 有效令牌可完成交易站大宗买入并扣减次数。
  - 私下交易大宗报价接受时要求买方令牌。
  - 有效买方令牌可完成私下大宗交易并绑定报价单。

## 风险与后续注意

- MVP 版本的令牌发放接口暂时允许玩家自助创建，后续需要改为管理员发放、审批或任务产出。
- 当前大宗令牌按成交时立即扣减，没有单独的预占机制；如果后续引入多阶段撮合交易，需要增加令牌冻结和释放流程。
- 当前仍未接入登录态/token，接口依赖 URL 中的 `userId`，后续需要统一认证鉴权。

- 或先实现前端联调页面：展示交易密码、令牌列表、交易站买卖和私下交易接受时的令牌输入。
