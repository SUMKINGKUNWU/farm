# 0005 - 功能分析与 MVP 边界

## 当前已有

当前已确定：

- 页面设计采用 `0004` 原型方向。
- 玩家初始资金为 10,000。
- 土地和牧场栏位最多各 16 个。
- 扩建下一个栏位所需资金翻倍。
- 交易功能必须先设置交易密码。
- 技术栈为 Vue 3 + Vite + Pinia + ECharts、Spring Boot、PostgreSQL、Redis。

## 本次完成

本次分析功能模块、MVP 边界、核心状态机、后台能力、风控点和后续开发优先级。

## 功能总览

项目可以拆成六个核心域：

- 账户域：用户、登录、钱包、交易密码、权限。
- 经营域：土地、牧场、扩建、种植、养殖、成长、收获。
- 资产域：金币、商品库存、令牌、资产流水。
- 交易域：交易站、私下交易、税收、大宗交易。
- 行情域：成交价、成交量、价格浮动、价格快照。
- 后台域：商品配置、税率配置、风控、审计、运营数据。

其中最重要的是资产域和交易域。只要涉及金币、库存、税收、令牌，都必须以后端事务和流水为准。

## 玩家端功能

### 注册与登录

MVP 必须包含：

- 用户注册。
- 用户登录。
- 登录态保持。
- 退出登录。
- 基础用户信息。

建议暂不做：

- 第三方登录。
- 手机短信登录。
- 实名认证。
- 复杂邀请体系。

### 钱包

规则：

- 新用户初始资金 10,000。
- 钱包余额只能由后端交易、扩建、收获出售、后台发放等业务改变。
- 所有余额变化必须写入资产流水。
- 金额使用整数最小单位，不能使用浮点数。

钱包流水类型：

- 初始发放。
- 商店购买。
- 扩建支出。
- 交易收入。
- 交易支出。
- 税费支出。
- 后台调整。

### 交易密码

规则：

- 玩家只要开始使用交易功能，就必须先设置交易密码。
- 交易密码用于交易站、私下交易、大宗交易、重要资产操作。
- 交易密码不能明文保存。
- 连续错误需要限制尝试次数。

MVP 流程：

```text
玩家点击交易功能
如果未设置交易密码 -> 弹出设置交易密码
设置成功 -> 允许继续交易
交易时输入交易密码
后端校验成功 -> 执行业务事务
```

建议交易密码适用范围：

- 交易站买入。
- 交易站卖出。
- 创建私下交易报价单。
- 接受私下交易报价单。
- 使用大宗令牌。
- 取消高价值订单。

## 经营系统

### 土地

规则：

- 土地用于种植作物。
- 土地有空闲、成长中、可收获状态。
- 每块土地同时只能有一个成长实例。
- 土地最多扩建到 16 个。

状态机：

```text
EMPTY -> GROWING -> READY -> HARVESTED -> EMPTY
```

关键防护：

- 成长中不能重复播种。
- 未成熟不能收获。
- 已收获不能重复收获。
- 收获必须在事务内完成状态变更和库存增加。

### 牧场

规则：

- 牧场栏位用于养殖动物。
- 牧栏有空闲、养殖中、可收获状态。
- 牧栏最多扩建到 16 个。
- 后续可以加入饲料、健康度、产出周期。

MVP 可简化为：

- 购买动物。
- 放入牧栏。
- 等待成熟或产出。
- 收获牧产品。

### 扩建

规则：

```text
最大土地数 = 16
最大牧栏数 = 16
下一次扩建费用 = 当前扩建费用 * 2
```

建议配置：

- 土地初始数量。
- 牧栏初始数量。
- 土地第一次扩建费用。
- 牧栏第一次扩建费用。
- 最大土地数。
- 最大牧栏数。

扩建事务：

```text
查询玩家当前栏位数
校验未达到 16
由服务端计算扩建费用
校验余额足够
扣除金币
新增栏位
写入资产流水
提交事务
```

## 商品与库存

### 商品类型

建议商品分为：

- 种子。
- 幼崽或动物。
- 饲料。
- 收获物。
- 令牌。
- 消耗道具。

MVP 必须至少包含：

- 小麦种子。
- 玉米种子。
- 鸡或鸡苗。
- 奶牛。
- 小麦。
- 玉米。
- 鸡蛋。
- 牛奶。
- 大宗交易令牌。

### 库存

库存规则：

- 每个玩家每种商品一条库存记录。
- 库存不能为负数。
- 冻结库存和可用库存要区分。
- 挂单或私下交易待确认时，需要冻结库存。

建议字段：

```text
user_id
item_id
available_quantity
locked_quantity
version
```

## 交易站

### MVP 交易站方案

建议 MVP 不做完整撮合引擎，先做系统价即时买卖。

流程：

```text
玩家选择商品
输入数量
系统按当前参考价计算金额
计算 3% 交易站税
输入交易密码
后端校验余额或库存
后端成交
写入成交记录、税收记录、资产流水
更新行情参考数据
```

优点：

- 开发更快。
- 可先验证玩法。
- 并发难度低于撮合订单簿。

缺点：

- 市场真实感弱。
- 玩家之间的供需关系较弱。
- 价格容易像系统商店。

### V1 交易站方案

V1 再加入：

- 买单。
- 卖单。
- 订单簿。
- 部分成交。
- 取消订单。
- 成交撮合。

撮合必须特别注意并发锁，否则容易出现重复成交。

## 私下交易

### MVP 私下交易

私下交易采用报价单模式。

流程：

```text
发起方选择对方玩家
选择商品和数量
填写金币价格
输入交易密码
系统冻结发起方商品
报价单进入待确认
接收方查看报价
接收方输入交易密码确认
系统校验接收方金币
扣私下交易税 5%
转移商品和金币
写入流水
报价单完成
```

状态机：

```text
DRAFT -> WAIT_ACCEPT -> SETTLING -> COMPLETED
                 |          |
                 v          v
              CANCELLED   FAILED
```

关键规则：

- 报价单创建后冻结商品。
- 接收方确认时冻结或扣除金币。
- 税费由服务端计算。
- 如果交易金额或数量达到大宗阈值，必须校验大宗令牌。
- 报价过期后自动释放冻结库存。

## 大宗交易

大宗交易触发条件：

- 数量达到阈值。
- 金额达到阈值。
- 商品被配置为大宗敏感商品。

令牌建议字段：

```text
token_id
user_id
allowed_item_type
single_trade_limit
total_limit
used_amount
remaining_uses
expires_at
status
```

MVP 可以简化为：

- 用户拥有令牌。
- 令牌未过期。
- 单笔交易使用 1 次。
- 使用后减少次数或标记已使用。

## 税收系统

默认规则：

- 交易站税率 3%。
- 私下交易税率 5%。
- 管理员可配置。

税费计算：

```text
税费 = 交易金额 * 税率
实际到账 = 交易金额 - 税费
```

建议金额向下或向上取整规则必须固定，不能前后不一致。

税收记录必须保存：

- 交易类型。
- 交易单号。
- 付款用户。
- 收款用户。
- 税率。
- 税额。
- 创建时间。

## 行情与价格

### MVP 价格方案

先做参考价浮动，不做复杂撮合。

建议公式：

```text
新价格 = 当前价格 * (1 + 成交量影响 + 买卖方向影响)
```

保护规则：

- 单笔成交对价格影响有限。
- 每小时涨跌幅有限。
- 每日涨跌幅有限。
- 成交量过低时价格回归基础价。
- 异常账号交易降低权重。

### V1 价格方案

加入：

- 最近成交量 VWAP。
- 买卖盘深度。
- 价格快照。
- K 线数据。
- 市场熔断。

## 后台管理

MVP 后台必须包含：

- 商品配置。
- 税率配置。
- 扩建费用配置。
- 大宗交易阈值配置。
- 用户查询。
- 交易记录查询。
- 资产流水查询。

建议 V1 增加：

- 风控事件。
- 管理员操作审计。
- 手动冻结用户。
- 手动冻结交易。
- 商品上下架。
- 活动配置。

后台所有修改必须写审计日志。

## 并发与一致性

必须使用数据库事务处理：

- 扩建。
- 播种。
- 收获。
- 交易站买卖。
- 私下交易创建。
- 私下交易确认。
- 私下交易取消。
- 令牌使用。

建议策略：

- 钱包使用行锁或乐观锁。
- 库存使用行锁或乐观锁。
- 土地和牧栏状态更新带版本号。
- 订单状态更新带版本号。
- Redis 可用于限流和短期幂等，但不能替代数据库事务。
