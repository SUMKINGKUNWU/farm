# 0056 后端交易站行情并发快照测试

## 当前已有

- 交易站已覆盖单线程买卖、税费、流水、价格快照和大宗令牌。
- 已补充并发卖出测试，验证库存不会被并发重复扣减。
- 已补充并发买入测试，验证钱包余额不会被并发重复扣款导致透支。

## 本次完成

- 在 `FarmExchangeApplicationTests` 新增 `concurrentMarketBuyWritesPriceSnapshotsWithinPriceBounds`。
- 模拟两个用户同时买入少量 `WHEAT`，让两笔交易都能成功成交。
- 断言成交量为 20、买入成交记录为 2、价格快照记录为 2。
- 断言当前价落在 26 到 27 之间，避免并发执行顺序导致测试不稳定。
- 断言快照价格仍在商品基础价允许区间内，没有低于半价或高于两倍价。
- 断言快照记录的 24 小时成交笔数处于合理范围，避免快照写入异常。

## 注意事项

- 当前 `MarketService` 的价格更新基于成交时读取到的 `current_price`，没有对商品价格行做显式行锁。
- 因此两笔并发买入后最终价可能是 26，也可能是 27，本测试先固定验证“成交和快照不丢、价格不越界”。
- 如果后续要保证每笔成交严格累积影响，需要在价格更新阶段引入商品行锁或乐观版本重试，再补更严格的最终价断言。

D:\maven\apache-maven-3.6.3\bin\mvn.cmd -s maven-settings.xml test

## 下次建议

- 优化 `MarketService` 行情价格更新逻辑，为 `items.current_price` 增加行级锁或乐观重试，保证并发价格影响严格累积。
- 增加对应回归测试，将两笔并发买入后的最终价稳定断言为 27。
